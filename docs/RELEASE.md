# Release Checklist

This project uses GitHub Actions to build `.apk` artifacts with an OpenWrt SDK.
A hosted apk feed is not part of the current release process.

## Before Building

- Confirm the public docs do not contain private deployment details.
- Confirm the package metadata in `luci-app-sing-box-config/Makefile` is correct.
- Run local static checks from [BUILD.md](BUILD.md).
- Review the default UCI example profile for placeholder values.

Run a local privacy scan for private router addresses, SSH deployment commands,
device-specific LuCI URLs, and local test-stand wording. The scan should not
report private deployment instructions in tracked public documentation.
`docs/private/` is ignored and must not be staged.

## Build Manually In Actions

To build an artifact without publishing a release:

```text
Actions -> Build OpenWrt APK -> Run workflow
```

Use inputs matching the target OpenWrt SDK:

- `openwrt_version`, for example `25.12.2`;
- `target`, for example `x86`;
- `subtarget`, for example `64`.

Download the generated artifact from the workflow run. It contains:

- `luci-app-sing-box-config_*.apk`;
- `SHA256SUMS`.

## Publish A GitHub Release

Create and push a version tag:

```sh
git tag v0.1.0
git push origin v0.1.0
```

The `Build OpenWrt APK` workflow builds the package and creates or updates the
GitHub Release for the pushed tag. It uploads the generated `.apk` and
`SHA256SUMS`.

## Smoke Test

Install the package on a compatible OpenWrt target:

```sh
apk add --allow-untrusted ./luci-app-sing-box-config_*.apk
```

After installation:

```sh
rm -rf /tmp/luci-indexcache /tmp/luci-modulecache
/etc/init.d/rpcd restart
/etc/init.d/uhttpd restart
/usr/libexec/sing-box-config-cli preview >/tmp/sing-box-config.preview.json
/usr/libexec/sing-box-config-cli validate
```

`validate` runs `sing-box check`, so it can fail when the profile still contains
placeholder server, UUID, Reality, or local rule-set values. Replace the example
profile values before treating a profile-level sing-box error as a package
installation failure.

## Publish

Create a GitHub Release and attach:

- `luci-app-sing-box-config_*.apk`
- checksums, if generated by the build environment

Suggested release notes:

```text
Initial public release of luci-app-sing-box-config.

Install on apk-based OpenWrt:
apk add --allow-untrusted ./luci-app-sing-box-config_*.apk
```

## Do Not Publish

Do not stage or attach:

- private router addresses;
- SSH deployment recipes for local test devices;
- private LuCI URLs;
- local recovery notes tied to a specific device;
- files under `docs/private/`.
